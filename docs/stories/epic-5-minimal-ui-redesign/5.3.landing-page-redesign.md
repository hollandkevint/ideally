# Story 5.3: Landing Page Redesign

## Status
Ready for Development

## Story
**As a** potential user visiting Thinkhaven for the first time,
**I want** a clean, compelling landing page that clearly communicates the product value,
**so that** I can quickly understand what Thinkhaven offers and decide to sign up.

## Acceptance Criteria

### Functional Requirements
1. **Sticky Navigation**: Logo left, nav links center, CTA buttons right - remains visible on scroll
2. **Hero Section**: Left-aligned headline "Think strategically. Build with clarity." with subheadline and two CTAs
3. **Social Proof Bar**: "Trusted by strategic thinkers" with company logos or user count
4. **Feature Cards**: 3 feature cards with icons, bold titles, and short descriptions
5. **Product Preview**: Interactive demo preview or screenshot of dual-pane interface
6. **Minimal Footer**: Essential links and copyright only

### Integration Requirements
7. Existing authentication flows work unchanged (signup/login CTAs)
8. New design uses existing shadcn/ui Button and Card components
9. Integration with existing Next.js 15.5 App Router maintained
10. Navigation preserves current routing structure

### Quality Requirements
11. Responsive design works on mobile, tablet, and desktop (320px - 1920px)
12. All interactive elements have proper hover and focus states
13. Page loads in < 2 seconds on 3G connection (Lighthouse)
14. Accessibility standards met (WCAG 2.1 AA)

### Technical Implementation Details

**Implementation Approach [Source: Original plan Section 2]:**
1. **Complete rewrite** of app/page.tsx (current landing is minimal)
2. **Pure white background** - no gradients or complex backgrounds
3. **Minimal shadows** - subtle elevation only on hover
4. **Icon library** - Use Lucide React for feature icons
5. **Animation** - Subtle scroll reveal (optional, framer-motion if needed)

**Visual Design Principles:**
- Generous whitespace (padding: 96px top/bottom for sections)
- Bold typography hierarchy (headings much larger than body)
- Single accent color (blue) - used sparingly
- Subtle borders (1px #e5e5e5)
- Clean card design (white with border, no heavy shadows)

**Responsive Breakpoints:**
- Mobile: < 768px (1 column, stacked layout)
- Tablet: 768px - 1024px (2 columns for features)
- Desktop: > 1024px (3 columns, full hero layout)

### Performance Requirements

- **Load Time**: < 2 seconds on 3G connection (Lighthouse)
- **Performance Score**: >= 90 (Lighthouse)
- **CLS**: < 0.1 (no layout shift)
- **Image Optimization**: Use Next.js Image component for screenshots

### Testing Requirements

**Testing Standards [Source: architecture/testing-strategy.md]:**
- E2E testing for authentication flow
- Responsive design validation
- Performance testing (Lighthouse)
- Accessibility audit (WCAG 2.1 AA)

**Specific Test Scenarios:**

1. **Navigation Testing:**
   - Verify sticky behavior on scroll
   - Test mobile hamburger menu functionality
   - Check CTA buttons navigate correctly
   - Validate logo click returns to top

2. **Authentication Flow:**
   - "Get started" CTA navigates to /signup
   - "Log in" button navigates to /login
   - Existing signup/login pages work unchanged

3. **Responsive Testing:**
   - Mobile (375px): Single column, stacked hero
   - Tablet (768px): 2-column features, adjusted hero
   - Desktop (1440px): 3-column features, full hero layout

4. **Performance Testing:**
   - Lighthouse audit (target: 90+ performance)
   - Page load time < 2s on slow 3G
   - No render-blocking resources
   - Images optimized and lazy-loaded

5. **Accessibility Testing:**
   - All interactive elements keyboard accessible
   - Proper heading hierarchy (h1 → h2 → h3)
   - Sufficient color contrast (4.5:1 minimum)
   - ARIA labels on navigation and CTAs
